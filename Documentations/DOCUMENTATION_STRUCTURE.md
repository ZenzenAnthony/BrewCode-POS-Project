# Documentation Structure Guide

This document explains the organization of documentation in the BrewCode POS Project.

## Documentation Organization

Project documentation is organized across multiple locations:

1. **Database Documentation** - Located in the `Database/` folder:
   - Design documents
   - SQL schema documentation
   - Menu-inventory relationship documentation
   - Database update summaries

2. **CSV Data Documentation** - Located in the `CSV/` folder:
   - CSV file structure and format
   - Sample data references
   - Import instructions

3. **Application Documentation** - Located in the `flutter_application_1/` folder:
   - Status reports
   - Improvements summary
   - Application READMEs
   - Integration plans

## Key Documentation Files

### Project-Level Documentation
- `README.md` - Main project overview
- `DOCUMENTATION_GUIDE.md` - Guide to finding documentation
- `CLEANUP_PLAN.md` - Project cleanup and maintenance

### Database Documentation
- `Database/COMPLETE_MENU_INVENTORY_RELATIONSHIPS.md` - Comprehensive list of menu-inventory relationships
- `Database/MENU_INVENTORY_ENHANCEMENT_SUMMARY.md` - Summary of database enhancements
- `Database/FINAL_IMPLEMENTATION_SUMMARY.md` - Final implementation details

### Application Documentation
- `flutter_application_1/STATUS_REPORT.md` - Current status of the Flutter application
- `flutter_application_1/IMPROVEMENTS_SUMMARY.md` - Planned improvements
- `flutter_application_1/database_integration_plan.md` - Plan for integrating with the database

## Note

This documentation structure was reorganized on May 22, 2025, to improve clarity and eliminate redundancy.
